Investigation

Existing Interactive Information Systems Research

Existing Interactive Information Systems

Data drives decision-making, efficiency, and innovation in Ireland(1). The Irish government promotes open data initiatives, enhancing transparency and evidence-based policymaking(2). Public databases like the Central Statistics Office (CSO) and Environmental Protection Agency (EPA) support climate policies, resource management, and public services (2). Open data improves healthcare, education, and infrastructure, helping public trust in government decisions(2). Irish datasets offer localised insights. Global datasets, such as the Global Trade Analysis Project (GTAP), enable international comparisons to address large-scale challenges(3).

My initial interest for this project was around the environmental impact of transport. Leading me to investigate the Transport Infrastructure Ireland (TII) Data Portal, a key transport repository, providing open access to national road and traffic data(4). This interactive system allows users to analyse and visualise transportation networks using various formats, including downloadable files, application programming interfaces (APIs), and real-time data(4). The EPA offers an interactive map tool to visualise and analyse environmental data such as water quality, land use, and protected areas(5).

As neither of these datasets satisfied my area of interest, this led me to research EV datasets to identify the most environmentally sustainable EVs available.

Research into the Chosen Area of Topic

Dataset Selection

I chose the EV_cars.csv dataset from Kaggle(6) for its comprehensive data on global EV models. The dataset includes data types such as price, acceleration, battery size, efficiency, range, and energy consumption. It also links to external resources for deeper insights about each EV model. This helped refine my project research question to solve:

"Which EV offers the best balance of affordability, efficiency, and performance for individuals or families based on living situation, household size, and income?"

Society Inpacts

Public access to the comprehensive information within the dataset EV_cars.csv from Kaggle(6) empowers consumers to make informed purchasing decisions, promotes transparency in the EV market, and supports research in efficiency and innovation in sustainable transportation. This enhances the Irish government’s open data initiatives(2).

Data analytics comparisons between EV models is an interactive tool to understand consumer behaviour, encourage eco-friendly EV adoption, reducing carbon emissions and fossil fuel dependence(7). For example, consumers can compare battery range and efficiency to choose an EV that suits their commuting needs(6). Providing accurate and transparent information helps increase public trust in government decisions and promotes greater EV adoption(8).

This dataset aligns closely with my interest in the environmental impact of transport.

Plan and Design

Project Overview

My project is a web-based interactive system with data dynamic visualisation, and analysis of a global EV dataset for consumers. Users can explore, filter, interpret data of EV price, acceleration, battery size, efficiency, range, and energy consumption, populate EV recommendations via a questionnaire based on their affordability and EV choice. The homepage details each page’s function, including a pop-up contact form of four data types, directing users to complete or exit, this data is stored to a separate database for retrieval, and this data is displayed in the ‘database results’ page.

Problems and Solutions

One major issue I encountered was duplicate forms and results appearing every time on the questionnaire submission. Instead of replacing old results, new ones stacked on the page, making it confusing for users. The problem was caused by how JavaScript handled form submissions, it appended new content instead of replacing the previous results. This affected usability and also impacted performance by unnecessarily increasing the number of elements on the page.

To fix this, I ensured old forms and results were cleared before adding new ones, preventing duplicates. Additionally, I introduced a submission flag to block multiple submissions at the same time, avoiding accidental re-submissions caused by users multiple clicking the button. These fixes improved UX, ensuring results update without clutter.

Analytics Algorithm

My structured analytics algorithm to process, analyse, and visualise EV data dynamically is:

  1. Load & Clean the Data
    • The dataset (EV_cars.csv) is loaded into a Pandas DataFrame.
    • Numeric values such as price, range, acceleration, and efficiency are converted to ensure proper data types.
    • Missing values are handled by filling them with median values to avoid errors in visualization.
  2. Determine the User's Theme Preference
    • The system checks whether the user has chosen light mode or dark mode.
    • The graph colours and styles are dynamically adjusted based on user preference.
  3. Generate Key Graphs Using Plotly
    • Scatter Plot (Price vs. Range) → Helps users compare EV affordability vs. battery range.
    • Line Scatter (Acceleration vs. Car Model) → Visualises EV performance based on acceleration performance 0-100.
    • Histogram (Price Distribution) → Shows how EV prices are distributed across different models.
  4. Format and Convert Graphs for Display
    • Each graph is converted into JSON format to be sent to the Flask app.
    • This ensures that the graphs can be dynamically rendered on the front-end without refreshing the page.
  5. Display Graphs in Flask
    • Flask retrieves the JSON data and renders the graphs in interactive visualisations.
    • Users can hover over points, zoom in, and analyse data to make informed EV choices.

Evaluation

My project successfully retrieves EV information from a dataset. Allowing users to search and filter relevant information and results, on all global EV’s available. Users can find the most suitable EV, potentially influencing their purchasing decisions based on their specific preferences of priority. The basic and advanced features required were implemented to provide users insights through dynamic visualisation tools. My project features visulisation analytic tools, for example, users can gain insights on the costs of battery consumption, the performance range of the battery, ranges of EV prices to suit affordability, and the EV acceleration if top speed performance is a priority. Users can input their preferences into a data collection questionnaire to dynamically present their personalised EV recommendations. Additionally, a database system was implemented to store and retrieve user details from a pop-up registration form. This pop-up registration form allows users to send contact information, ensuring a more personalized experience in a real-world situation.

During development, I originally planned to use Streamlit due to its rapid prototyping capabilities. It allowed me to create a working UI, but became too restrictive as I needed to have greater control over interactivity and UX. Streamlit’s session handling was limited, and I struggled with implementing complex filtering and data persistence. I decided to switch to Flask, which provided greater flexibility, allowing me to handle form submissions, AJAX requests, and database interactions more efficiently. This design change ultimately improved performance and scalability, and more user- friendly.

Some ideas I have for my project in the future, would be to use machine learning to create personalized recommendations. Integrating machine learning could retrieve, store and display past users driving habits, driving patterns, for example city vs long distance driving. Incorporating API’s live data or from manufacturer websites could provide up-to-date results for users, making the website a viable solution tool to launch.
